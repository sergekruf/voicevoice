## [1.1.1] — 2026-05-30

### Изменено
- **Parakeet TDT v3 — теперь движок по умолчанию.** После теста на реальной русской речи: очень быстро, качество распознавания не хуже WhisperKit. Существующие пользователи сохраняют свой выбор; новый дефолт — только для свежих установок. WhisperKit остаётся доступен в Настройки → Движок (чуть лучше с пунктуацией, шире поддержка языков). Снят ярлык «экспериментально».
- **Всплывающие уведомления переехали в правый верхний угол** (стандартное место уведомлений macOS) — раньше висели внизу по центру и закрывали поле ввода в чатах (Claude Desktop, MAX, Telegram). Индикатор записи (микрофон) остался снизу по центру — это функциональный элемент, а не уведомление. Время показа без изменений.

## [1.1.0] — 2026-05-30

### Добавлено
- **Отмена диктовки по Esc.** Нажатие `Esc` во время записи отменяет диктовку без распознавания и вставки — защита активного поля от случайного/искажённого ввода. Транзиентные `NSEvent`-мониторы ставятся только на время записи (`AppController.cancelDictation`).
- **Прогресс загрузки модели.** На первом запуске ~632 МБ теперь скачиваются с реальным процентом (`Transcriber.buildPipeline` через `WhisperKit.download(progressCallback:)`) вместо «зависшей» надписи — состояние `.downloading(progress:)` раньше существовало в UI, но никогда не выставлялось. Есть fallback на прежний config-managed путь (офлайн с уже скачанной моделью).
- **Редактируемый блоклист галлюцинаций.** Список фраз-«титров» вынесен в Настройки → «Фильтр галлюцинаций» (`AppSettings.hallucinationBlocklist`, кнопка «Сбросить»). Технические kill-токены (DimaTorzok и т.п.) остаются в коде.
- **Опциональный движок Parakeet TDT v3** (через FluidAudio, Настройки → Движок). ~5× быстрее WhisperKit, ~66 МБ RAM, поддерживает русский; модель (~600 МБ) скачивается только при выборе. Дефолт остаётся WhisperKit. Слабее с пунктуацией — компенсируется пост-обработкой. `ParakeetTranscriber`, `AppSettings.sttEngine`. ⚠️ Экспериментально — сравните качество на своей речи.
- **Распознавание во время записи (eager-стриминг)** (новый тоггл «Распознавать во время записи» в секции «Модель распознавания», по умолчанию **ON**). Законченные VAD-чанки распознаются в фоне, пока пользователь ещё говорит; на отпускании Fn остаётся распознать только короткий хвост — для длинных диктовок текст появляется почти мгновенно. Границы чанков совпадают с обычным режимом (та же логика реза по тишине), меняется только момент распознавания. `Transcriber.startStreaming/finishStreaming`, снапшот буфера через `AudioRecorder.currentSamples()`.
- **Блоклист галлюцинаций Whisper.** На тишине/шуме Whisper выдаёт заученные с YouTube «титры» («Субтитры сделал DimaTorzok», «Продолжение следует», «Подписывайтесь на канал», «Редактор субтитров…» и т.п.). `Transcriber.stripHallucinations` детерминированно выкидывает их — но только при совпадении с ЦЕЛЫМ предложением (или наличии kill-токена вроде `dimatorzok`), так что настоящая речь («Спасибо за внимание, коллеги») не страдает. Список расширяется из логов.
- **Ручное добавление правок в словарь.** Кнопка «Добавить» в «Словарь правок…» открывает форму wrong→right→контекст; запись сразу активна (`CorrectionStore.addManual`, seeds confirmedCount=3). Раньше словарь пополнялся только автообучением или импортом JSON.
- **Авто-исправление знаков в конце предложений** (новый тоггл «Исправлять знаки в конце предложений» в секции «Распознавание», по умолчанию **ON**). Простые русские правила компенсируют интонационные ошибки Whisper-turbo 4-bit:
  - Частица «ли» в предложении → терминатор становится «?».
  - Начало с вопросительного слова (что / где / когда / почему / куда / откуда / зачем / кто / сколько / разве / неужели), с допустимыми дискурсивными «А / Ну / Так / И» → «.» меняется на «?».
  - Длинное предложение (≥ 5 слов) без вопросительных маркеров, оканчивающееся на «?» → меняется на «.».
  - «!» и слова «как / какой / который» не трогаются — слишком много восклицаний на их базе.
- **Авто-форматирование текста** (новый тоггл «Авто-форматирование (списки, абзацы)» в секции «Распознавание», по умолчанию **OFF**, opt-in):
  - Перечисления «первое… второе…» / «во-первых… во-вторых…» (2+ подряд) → нумерованный список с переводами строк (`1. …\n2. …`).
  - Sequence-маркеры «сначала… потом… затем…» (3+ разных) → нумерованный список.
  - «список покупок:» / «нужно купить:» / «перечислю:» / «пункты:» / «три задачи:» / «несколько вещей:» + 3+ элемента через запятую → маркированный список.
  - Action-verb списки: «надо купить хлеб, молоко, масло, яйца» (action-глагол + 3+ запятых) → маркированный список с заголовком из всей префиксной части предложения.
  - Фразы «новый абзац» / «с новой строки» / «новая строка» / «абзац» (как самостоятельная фраза, ограниченная пунктуацией) → `\n\n`.
  - Выключен по умолчанию, потому что вставляет переводы строк — это ломает узкие однострочные поля ввода. Включить можно в Настройки → Распознавание.

### Изменено
- **Чанкинг режет только по «настоящей» тишине (≥300 мс).** `Transcriber.findSilenceCut` теперь принимает точку реза только если найденная тишина ≥ `minSilenceFrames` (3 фрейма EnergyVAD по 0.1 с) — иначе режет по целевым 12 с. Раньше резало и по 100-мс провалу энергии, что часто было паузой ВНУТРИ слова. Без новых зависимостей; полноценный Silero VAD при необходимости — отдельный шаг.

### Исправлено
- **Длинные диктовки плотной речи (25–60+ с) обрывались на полуслове.** Корень — два жёстких ограничения WhisperKit 0.18.0: декодер клипит `sampleLength` до `Constants.maxTokenContext - 1 = 223` (`TextDecoder.swift: loopCount = min(sampleLength, maxTokenContext - 1)`), а встроенный VAD-чанкер вообще не режет аудио ≤ 30 с (`AudioChunker.swift: if audioArray.count <= maxChunkLength { return [single] }`). Итог: одна плотная фраза 25-30 с упиралась в 223 токена и обрывалась посередине слова. Правка 224→448 из 1.0.2 не работала — WhisperKit всё равно клипил внутри. **Фикс**: режем аудио сами в `Transcriber.preChunk(audio:)` до передачи в `pipe.transcribe()` — куски ≤ 12 с по границам тишины (через публичный `EnergyVAD` + `findLongestSilence`), каждый чанк декодируется отдельно, результаты склеиваются.
- **Распознанный текст оставался в буфере после успешной вставки** в приложениях без AX-доступа (Claude Desktop, MAX, Termius) — следующий ⌘V повторно вставлял продиктованное. Теперь ветка `axUnreadable` вызывает `finalizeAfterPaste` симметрично с `.editable`: при выключенном «Оставлять текст в буфере» буфер восстанавливается.
- **Пустой результат при включённом punctuation-prompt на некоторых моделях** (`large-v3-turbo` full precision): промпт ломал декодер (мгновенный EOT). Добавлен empty-prompt fallback — при пустом результате перераспознаём без промпта.

## [1.0.2] — 2026-05-25

### Исправлено
- **Длинные диктовки (40–60+ с) теряли куски в середине.** Без явного `chunkingStrategy` WhisperKit падал в встроенный seek-loop: при провале одного 30-секундного окна (temperature-fallback exhaustion или `noSpeechThreshold`) seek прыгал вперёд на полные 30 секунд, теряя весь контент окна. Включён `chunkingStrategy: .vad` — аудио режется по тишине через `VADAudioChunker`, провал одного чанка не влияет на соседей. (**См. также [1.1.0]** — для плотной речи без длинных пауз `.vad` оказался недостаточен, проблема была дополнительно закрыта pre-chunkingом до WhisperKit.)
- **Лимит выходных токенов**: `sampleLength` 224 → 448. ⚠︎ **Эта правка не работала в рантайме**: WhisperKit внутри клипит `sampleLength` до `Constants.maxTokenContext - 1 = 223` (см. фикс в [1.1.0]). Значение возвращено к 224.

### Изменено
- **Дашборд: счётчики «за всё время» вместо «последние 200»**. Таблица истории капится в 200 записей, и прежний `HistoryStore.stats()` считал по триммнутой таблице — счётчик «Всего расшифровок» упирался в 200 и не рос. Добавлены `AppStorage`-счётчики `lifetimeRecordsCount` / `lifetimeCharactersCount` / `lifetimeAudioSeconds` / `lifetimeProcessingMs` / `firstRecordAt`, инкрементируемые на каждой диктовке. При первом запуске на этой версии — одноразовая миграция: lifetime-поля заполняются из текущего содержимого БД (бэкфилл из имеющихся ≤ 200 записей).

## [1.0.0] — 2026-05-22

Первый публичный релиз.

### Добавлено
- **Hotkey-диктовка** через удержание `Fn`, правого `⌥ Option` или `Caps Lock` (выбор в настройках).
- **Локальный Whisper** через [WhisperKit](https://github.com/argmaxinc/WhisperKit) — модель `large-v3-turbo` 4-bit на Apple Neural Engine, ~10× быстрее реального времени на M4.
- **Авто-обучение словаря** правок: ~5 минут после успешной вставки watcher следит за фокусным полем через AX и запоминает пары `wrong → right`.
- **Fuzzy-matching** словаря (Levenshtein с настраиваемым порогом).
- **Edit & Learn** — окно ручной правки распознанного текста с записью в словарь. Доступно из HUD для приложений без AX-доступа (Bitrix24, Max, Slack, Termius…).
- **Трёхуровневая вставка**: CGEvent ⌘V → AppleScript → AXUIElement direct write.
- **TransientType-маркер** на промежуточные записи в буфер — клипборд-менеджеры (Maccy, Paste, PasteNow, Raycast) их игнорируют.
- **Нормализация чисел** — числовые формы из речи преобразуются в цифры, лишние пробелы и точки убираются.
- **Авто-эмодзи** (opt-in) — один уместный смайл по триггер-словам: «спасибо» → 🙏, «поздравляю» → 🎉, «хаха» → 😄.
- **Тихий режим** — скрывает все HUD'ы и тосты, кроме индикатора записи.
- **Подсказки-`?`** возле каждой настройки с пружинной анимацией и кастомным popover'ом.
- **История последних 200 распознаваний** + словарь правок с фильтрами.
- **Privacy-by-default** — ноль телеметрии, ноль облака.

### Стек
- Swift 6 / SwiftUI / AppKit
- WhisperKit + CoreML/ANE
- GRDB (SQLite-обёртка)
